# GitHub Pages — критерии выпуска академической миграции

## Цель

Опубликовать статический сайт `agent-axiom/parallel-worlds` вместе с воспроизводимым академическим аудитом, не выдавая 37 унаследованных линий за рецензированные.

## Критерии приёмки

- `npm test`, `bash scripts/validate.sh` и `git diff --check` завершаются успешно.
- `academic-audit.json` входит в Pages-артефакт и доступен по `https://agent-axiom.github.io/parallel-worlds/academic-audit.json`.
- Аудит содержит: 62 линии, 25 `reviewed`, 37 `legacy`, 0 блокирующих ошибок, 42 предупреждения.
- Покрытие аудита составляет 69 периодов и 50 событий с источниками и метаданными датировки.
- Фильтры `reviewed` и `legacy` работают и сохраняются в URL.
- Русская, английская и китайская версии показывают карточки доказательств, альтернативы и точные ссылки.
- Границы Старовавилонской и Аршакидской хронологий читаемы на десктопе и экране шириной 390 px.
- GitHub Actions публикует ветку `main`, а главная страница и аудит возвращают HTTP 200.

## Локальная проверка

```bash
npm test
bash scripts/validate.sh
git diff --check
npm run serve
```

После запуска проверить `http://localhost:8080/?type=reviewed`, переключить RU/EN/ZH и открыть карточки Вавилонии, Персии, Рима и Византии.

## Выпуск

1. Слить проверенную ветку в `main` без потери пользовательских изменений.
2. Отправить `main` в `agent-axiom/parallel-worlds`.
3. Дождаться успешного workflow `Deploy GitHub Pages`.
4. Проверить опубликованный сайт и `academic-audit.json`.

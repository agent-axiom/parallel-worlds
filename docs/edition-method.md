# Методика подарочного издания

Этот документ описывает Release A — проверяемый производственный фундамент подарочного издания «Параллельные миры». Он связывает академические данные сайта, будущий печатный макет, медиареестр и постоянные QR-маршруты, но не заменяет редактуру, лицензирование изображений, допечатную подготовку и физическую приёмку тиража.

## Неизменяемая структура

Канонический манифест `edition-data.js` фиксирует:

- альбомный формат 300 × 240 мм;
- русское печатное издание с RU / EN / ZH сопровождением на сайте;
- 288 полос: 16 вводных, 12 окон по 16 полос, 8 интерлюдий по 4 полосы и 48 полос аппарата;
- минимальный тираж 500 экземпляров и запрос расчётов для 500 / 750 / 1000;
- 12 опорных исторических окон от 18 000 года до н. э. до 1450 года н. э.

Изменение любого из этих параметров требует новой версии манифеста и отдельного редакционного решения.

## Академический гейт

В печатный отбор входят только записи со статусом `reviewed`. Для каждого окна требуется не менее шести активных проверенных линий из трёх регионов; у выбранных записей должны быть собственные поля датировки и точные источники уровня A или B.

`edition-audit.json` различает два класса проблем:

- `blockingIssues` — структурная ошибка данных или схемы; сборка должна остановиться;
- `readinessGaps` — честно зафиксированная нехватка проверенных материалов или разрешённых изображений; код может публиковаться, но следующий производственный релиз не допускается.

Поле `releaseReady` становится `true` только после закрытия обоих классов. Для Release A значение `releaseReady: false` ожидаемо; оно должно сохраняться до завершения полного Release C.

## Единый медиареестр

Каждое изображение получает постоянный `mediaId`. Публичная запись содержит безопасные кредиты, академические связи, контрольную сумму мастера, локальные web-производные, подписи и alt-тексты RU / EN / ZH, а также признак реконструкции и её основание.

Закрытый производственный архив хранит исходные мастера, договоры, счета, контактные данные правообладателей и иные непубличные условия. Путь вида `private://…` или `archive://…` является непрозрачной ссылкой, а не публичным файлом.

Права `print` и `web` проверяются независимо. Наличие одного разрешения не подразумевает другого. Для печати запись должна явно разрешать не менее 500 экземпляров; для сайта требуются отдельный статус `web: cleared`, срок и территория. Неясный статус исключает файл из соответствующего канала.

Документальное изображение и реконструкция визуально и семантически различаются. Реконструкция без описанного основания не публикуется и никогда не используется как доказательство.

## Постоянные QR-маршруты

Печатные ссылки используют только относительные стабильные пути `/go/window-NN/` и, после появления разрешённых объектов, `/go/window-NN/mediaId/`. Генератор не принимает внешние redirect-цели, а каждая страница содержит статическую ссылку на случай отключённого JavaScript.

Release A проверяет маршруты на GitHub Pages. Для тиража нужен контролируемый издателем домен, зафиксированный отдельным подписанным release manifest; адрес `github.io` не является финальным тиражным доменом.

Прямой QR открывает веб-компаньон пассивно: карта и режиссёрское путешествие не запускаются автоматически. Русский выбран по умолчанию, английский и китайский доступны через переключатель сайта. Критически важный смысл не переносится только за QR.

## Воспроизводимая проверка

Пересобрать детерминированные артефакты:

```bash
node scripts/build-edition-audit.mjs
node scripts/build-companion-routes.mjs
```

Проверить весь репозиторий без изменения закоммиченных audit-файлов:

```bash
npm run validate
```

Валидация проверяет синтаксис модулей, свежесть академического и edition-аудита, точное соответствие дерева QR-маршрутов, тесты, относительные пути и полный состав Pages-артефакта.

## Границы Release A

Release A предоставляет схему, аудит, маршруты и доступный веб-вход. Он не содержит лицензированных печатных мастеров, окончательного русского текста 288 полос, верстки PDF/X-4, профиля выбранной типографии, цветопробы, переплётного макета или сигнального экземпляра.

Следующий Release B создаёт один законченный 16-полосный тестовый раздел и соответствующую web-галерею на реальных разрешённых изображениях. Полный Release C начинается только после утверждения тестового раздела, бюджета прав и производственной сметы.
